\chapter{Introdução}

\noindent O planeamento de compras continua a exigir que os consumidores consultem múltiplas fontes de informação para comparar preços, verificar promoções ou organizar listas de produtos. Este processo torna-se frequentemente demorado e ineficiente, especialmente quando envolve várias lojas ou supermercados.

\noindent Este projeto propõe o desenvolvimento de uma plataforma digital que centraliza informação de produtos e preços, permitindo aos utilizadores criar listas de compras inteligentes, comparar preços entre diferentes lojas e receber alertas quando ocorrem alterações relevantes.

\noindent A solução será disponibilizada através de duas interfaces principais: uma aplicação móvel e uma aplicação web.

\noindent A aplicação móvel será utilizada no contexto real de compras e permitirá aos utilizadores criar e gerir listas de forma simples, consultar e comparar preços entre diferentes lojas e visualizar preços próximos com base na localização do utilizador. Adicionalmente, os utilizadores poderão receber notificações quando um produto sofrer uma redução de preço e identificar rapidamente produtos através da leitura de códigos de barras ou códigos QR com a câmara do dispositivo.

\noindent A aplicação web permitirá uma preparação mais confortável das listas de compras num computador, bem como a consulta de comparações de preços entre diferentes lojas e localizações, oferecendo uma visão mais abrangente da informação disponível.

\noindent Um dos elementos centrais da plataforma será a georreferenciação de preços. O mesmo produto poderá ter valores diferentes consoante a loja e a localização geográfica, permitindo ao sistema apresentar ao utilizador as opções mais vantajosas com base na sua proximidade.

\noindent Desta forma, a aplicação pretende simplificar o processo de planeamento de compras e ajudar os utilizadores a tomar decisões mais informadas, poupando tempo e dinheiro.

\section{Contexto e Motivação}

\label{sec:1.1}

Nos últimos anos, a necessidade de otimizar o processo de planeamento de compras tem vindo a aumentar, impulsionada pela crescente diversidade de produtos, lojas e promoções disponíveis. Atualmente, os consumidores são frequentemente obrigados a consultar múltiplas fontes de informação para comparar preços, verificar campanhas promocionais e organizar listas de compras, o que torna este processo moroso e pouco eficiente. Esta realidade é particularmente evidente quando envolve diferentes superfícies comerciais, como supermercados e lojas físicas, onde os preços podem variar significativamente.

Este projeto foca-se especificamente no desenvolvimento de uma plataforma digital integrada que permita centralizar informação de produtos e preços, simplificando o processo de tomada de decisão do utilizador. A solução proposta combina uma aplicação móvel e uma aplicação web, permitindo não só a criação e gestão de listas de compras, mas também a comparação de preços entre diferentes lojas e localizações. Adicionalmente, são introduzidas funcionalidades como alertas de alteração de preços, leitura de códigos de barras e georreferenciação, proporcionando uma experiência mais inteligente e adaptada ao contexto real de compra.

\subsection{Âmbito do Problema}

A gestão eficiente de compras apresenta vários desafios que vão além da simples criação de listas de produtos. Um dos principais problemas reside na dispersão da informação, uma vez que os preços dos produtos variam consoante a loja e a localização geográfica, obrigando o utilizador a realizar múltiplas consultas para encontrar a melhor opção. Esta dificuldade é reforçada pelo facto de muitas soluções existentes se focarem apenas em contextos de comércio eletrónico, não respondendo adequadamente às necessidades de compras em lojas físicas.

Outro desafio relevante prende-se com a falta de integração entre funcionalidades. Aplicações atuais tendem a oferecer apenas gestão de listas ou comparação de preços de forma isolada, não existindo uma solução unificada que combine ambas de forma eficiente. Para além disso, a inexistência de mecanismos automáticos de alerta para variações de preço limita a capacidade do utilizador de aproveitar oportunidades de poupança em tempo útil.

Neste contexto, o problema a abordar passa pela criação de um sistema que integre, de forma coerente e escalável, a gestão de produtos, preços e lojas, garantindo simultaneamente desempenho, consistência de dados e relevância da informação apresentada ao utilizador. A introdução de georreferenciação e notificações em tempo real surge como elemento diferenciador, permitindo apresentar opções mais vantajosas com base na proximidade e nas preferências do utilizador.

\section{Abordagem}
\label{sec:1.3}

Para alcançar os objetivos definidos, foi concebido e desenvolvido um sistema distribuído composto por uma aplicação móvel, uma aplicação web e uma infraestrutura de backend, recorrendo a tecnologias modernas amplamente adotadas, de forma a garantir usabilidade, escalabilidade e facilidade de manutenção.

A solução proposta segue uma arquitetura orientada a serviços, na qual o backend se encontra dividido em vários serviços responsáveis por domínios específicos, como produtos, preços, lojas e utilizadores. Estes serviços são disponibilizados através de APIs REST, permitindo uma comunicação eficiente entre os diferentes componentes e promovendo uma clara separação de responsabilidades.

A aplicação móvel, desenvolvida com recurso a React Native, foi concebida para utilização em contexto real de compra, disponibilizando funcionalidades como gestão de listas, comparação de preços, leitura de códigos de barras e receção de notificações em tempo real. Por sua vez, a aplicação web, desenvolvida em React, oferece uma interface mais completa para preparação e análise das compras, tirando partido de um ambiente de utilização mais confortável.

A gestão de dados é assegurada através de uma base de dados relacional, garantindo integridade e eficiência nas operações de consulta, particularmente em cenários onde um mesmo produto pode apresentar preços distintos consoante a loja e a localização. A incorporação de mecanismos de georreferenciação permite ao sistema apresentar informação contextualizada, baseada na proximidade do utilizador.

Adicionalmente, o sistema integra um mecanismo de notificações push, possibilitando o envio de alertas relativos a alterações de preços, contribuindo para uma maior proatividade na tomada de decisão por parte do utilizador. O processo de desenvolvimento é suportado por pipelines de CI/CD, que permitem automatizar tarefas de construção, teste e deploy, assegurando a qualidade e consistência do software.

Em conjunto, esta abordagem permite o desenvolvimento de uma solução robusta, escalável e centrada no utilizador, capaz de responder de forma eficaz aos desafios identificados no domínio do planeamento de compras.

\section{Objetivos}
\label{sec:1.4}

Os objetivos principais do projeto ShopISEL são:

\begin{enumerate}
    \item Desenvolver uma plataforma integrada que permita ao utilizador criar e gerir listas de compras, consultar preços de produtos em múltiplas lojas e receber alertas de descida de preço;
    \item Implementar um sistema de recolha automática de dados de produtos e preços de insígnias de retalho portuguesas, sem dependência de APIs proprietárias;
    \item Disponibilizar a plataforma através de uma aplicação móvel (Android e iOS) e de uma aplicação web, com experiências de utilização adequadas a cada contexto;
    \item Integrar georreferenciação de lojas, permitindo ao utilizador identificar as opções mais vantajosas na sua proximidade geográfica;
    \item Garantir a segurança e privacidade dos dados do utilizador através de um sistema de autenticação centralizado baseado em padrões abertos (OIDC / OAuth 2.0);
    \item Assegurar a qualidade e a entrega contínua do sistema através de pipelines de \ac{CI/CD} automatizadas.
\end{enumerate}

\section{Requisitos do Sistema}
\label{sec:1.5}

Os requisitos do sistema foram definidos com base nas necessidades identificadas no âmbito do projeto e orientaram as decisões de arquitetura e implementação.

\subsection{Requisitos Funcionais}
\label{sec:1.5.1}

\begin{enumerate}
    \item O sistema deve permitir ao utilizador criar, consultar, atualizar e eliminar listas de compras pessoais;
    \item O sistema deve apresentar preços atualizados de produtos em múltiplas lojas e insígnias;
    \item O sistema deve notificar o utilizador quando o preço de um produto favorito descer;
    \item O sistema deve suportar a identificação de produtos através da leitura de códigos de barras pela câmara do dispositivo móvel;
    \item O sistema deve apresentar lojas ordenadas por proximidade geográfica com base na localização do utilizador;
    \item O sistema deve autenticar e autorizar utilizadores, garantindo o isolamento de dados entre contas distintas;
    \item O sistema deve recolher automaticamente dados de produtos e preços de lojas de retalho, sem dependência de APIs proprietárias.
\end{enumerate}

\subsection{Requisitos Não Funcionais}
\label{sec:1.5.2}

\begin{enumerate}
    \item \textbf{Segurança}: a autenticação deve seguir padrões abertos (\ac{OIDC} / OAuth 2.0), e os dados de sessão devem ser armazenados de forma cifrada nos dispositivos;
    \item \textbf{Escalabilidade}: a arquitetura deve permitir a adição de novos serviços e insígnias sem impacto nos componentes existentes;
    \item \textbf{Disponibilidade}: os serviços de backend devem estar continuamente disponíveis, com deployment automático em plataformas de cloud;
    \item \textbf{Manutenibilidade}: cada serviço deve ter o seu ciclo de build, teste e deployment independente, suportado por pipelines de \ac{CI/CD};
    \item \textbf{Portabilidade}: as aplicações cliente devem estar disponíveis em plataformas web e móveis (Android e iOS).
\end{enumerate}
