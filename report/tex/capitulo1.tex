\chapter{Introdução}

O ShopISEL é uma plataforma digital de apoio ao planeamento de compras, que centraliza informação de produtos e preços de múltiplas insígnias de retalho. A plataforma permite criar e gerir listas de compras, comparar preços entre lojas, receber alertas de descida de preço e identificar as opções mais vantajosas com base na localização do utilizador. A solução é disponibilizada através de uma aplicação móvel, orientada ao contexto real de compra, e de uma aplicação web, para preparação e análise num ambiente mais confortável.

\section{Contexto e Motivação}
\label{sec:1.1}

O processo de planeamento de compras é frequentemente moroso: os preços variam entre lojas e promoções, obrigando o utilizador a consultar múltiplas fontes para tomar uma decisão informada. As soluções existentes tendem a focar-se num único domínio — gestão de listas ou comparação de preços — sem os integrar numa experiência coesa. Acresce que a maioria das plataformas de comparação de preços se destina ao comércio eletrónico, não cobrindo adequadamente as lojas físicas de supermercado.

O ShopISEL procura colmatar esta lacuna, combinando gestão de listas, comparação de preços entre insígnias, alertas automáticos e georreferenciação de lojas numa única plataforma integrada.

\section{Abordagem}
\label{sec:1.2}

A solução segue uma arquitetura orientada a serviços, com o backend dividido em serviços independentes por domínio (listas, produtos, preços, lojas, contas), expostos através de \ac{API}s REST. A recolha automática de dados é assegurada por scrapers dedicados por insígnia, sem dependência de \ac{API}s proprietárias. A autenticação é centralizada no Keycloak, e as notificações push são entregues via Firebase Cloud Messaging. Todo o sistema é containerizado com Docker e suportado por pipelines de \ac{CI/CD} com GitHub Actions.

\section{Objetivos}
\label{sec:1.3}

Os objetivos principais do projeto ShopISEL são:

\begin{enumerate}
    \item Desenvolver uma plataforma integrada para criação e gestão de listas de compras, consulta de preços em múltiplas lojas e alertas de descida de preço;
    \item Implementar recolha automática de dados de produtos e preços de insígnias de retalho portuguesas, sem dependência de \ac{API}s proprietárias;
    \item Disponibilizar a plataforma em aplicação móvel (Android e iOS) e aplicação web;
    \item Integrar georreferenciação de lojas para apresentar as opções mais próximas e vantajosas;
    \item Garantir autenticação segura baseada em padrões abertos (\ac{OIDC} / OAuth 2.0);
    \item Assegurar qualidade e entrega contínua através de pipelines de \ac{CI/CD} automatizadas.
\end{enumerate}

\section{Requisitos do Sistema}
\label{sec:1.4}

\subsection{Requisitos Funcionais}
\label{sec:1.4.1}

\begin{enumerate}
    \item O sistema deve permitir criar, consultar, atualizar e eliminar listas de compras pessoais;
    \item O sistema deve apresentar preços atualizados de produtos em múltiplas lojas e insígnias;
    \item O sistema deve notificar o utilizador quando o preço de um produto favorito descer;
    \item O sistema deve suportar a identificação de produtos por leitura de código de barras;
    \item O sistema deve apresentar lojas ordenadas por proximidade geográfica;
    \item O sistema deve autenticar e autorizar utilizadores, garantindo isolamento de dados entre contas;
    \item O sistema deve recolher automaticamente dados de produtos e preços sem dependência de \ac{API}s proprietárias.
\end{enumerate}

\subsection{Requisitos Não Funcionais}
\label{sec:1.4.2}

\begin{enumerate}
    \item \textbf{Segurança}: autenticação com padrões abertos (\ac{OIDC} / OAuth 2.0) e dados de sessão cifrados;
    \item \textbf{Escalabilidade}: adição de novos serviços e insígnias sem impacto nos componentes existentes;
    \item \textbf{Disponibilidade}: deployment automático em plataformas de cloud com alta disponibilidade;
    \item \textbf{Manutenibilidade}: ciclo de build, teste e deployment independente por serviço;
    \item \textbf{Portabilidade}: disponível em plataformas web e móveis (Android e iOS).
\end{enumerate}
